Enhanced Object Literals

Next >>

In this lesson, I learnt about using enhanced object literals to make objects look cleaner and more readable. With this, we reduce redundancy in key/value pair naming and declaration of functions are much cleaner.

Code:

  1. let app = {};
  2.  
  3. app.createBookShop = (inventory) => {
  4.     return {
  5.         /*
  6.         * In ES6 when defining an object and a property has 
  7.         * the same name as the variable, we just pass the variable name.
  8.         */ 
  9.         inventory, 
  10.         /*
  11.         * Function properties are also defined without the function
  12.         * keyword and colon. This allows for the use of lexical 'this'.
  13.         */
  14.         inventoryValue() {
  15.             return this.inventory.reduce((total, book) => total + book.price, 0);
  16.         },
  17.         priceForTitle(title) {
  18.             return `USD ${this.inventory.find(book => book.title === title).price}`;
  19.         }
  20.     }
  21. };
  22.  
  23. const inventory = [
  24.     { title: 'Harry Potter', price: 10 },
  25.     { title: 'Eloquent Javascript', price: 15 }
  26. ];
  27.  
  28. const book_shop = app.createBookShop(inventory);
  29. let book_title = 'Harry Potter';
  30.  
  31. console.log(`Book shop inventory value result: ${book_shop.inventoryValue()}`);
  32. console.log(`Book shop title price for '${book_title}' result: ${book_shop.priceForTitle(book_title)}`);
