In this lesson, I learnt about the use of const and let in variable naming. The difference between const and let variables is that const variables are constants and do not change in value.

Code:

  1. /*
  2. * A variable is a “named storage” for data. In ES6 we use const
  3. * and let to declare variables. 'let' is used to declare variables
  4. * that can change in value at any given time. For 'const', these variables are
  5. * constant and cannot be changed.
  6. */
  7. const name = 'John Doe';
  8. let title = 'Software Engineer';
  9. let hourly_wage = 40;
  10.  
  11. console.log('Name: ' + name + ', Title: ' + title + ', Hourly Wage: ' + hourly_wage);
  12.  
  13. // name = 'Peter Doe'; Here we cannot change the value of a constant variable
  14. title = 'Senior Software Engineer';
  15. hourly_wage = 45;
  16.  
  17. console.log('Name: ' + name + ', Title: ' + title + ', Hourly Wage: ' + hourly_wage);
  18.  
  19. /*
  20. * There are two limitations on variable names in JavaScript:
  21. * 1. The name must contain only letters, digits, or the symbols $ and _.
  22. * 2. The first character must not be a digit.
  23. *
  24. * We can declare multiple variables by chaining them like so:
  25. */
  26. let user = 'John',
  27.     age = 25,
  28.     message = 'Hello';
  29.  
  30. /*
  31. * There is a widespread practice to use constants as aliases for difficult-to-remember
  32. * values that are known prior to execution. Such constants are named using capital letters and underscores.
  33. * 
  34. * For example, we can declare constant variables for color hexadecimal values:
  35. */
  36. const COLOR_RED = '#FF0000';
  37. const COLOR_GREEN = '#00FF00';
  38. const COLOR_BLUE = '#0000FF';
  39. const COLOR_ORANGE = '#FF7F00';
